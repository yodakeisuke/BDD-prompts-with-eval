# Question Patterns Reference

## Discovery Questions (発見の質問)

### Purpose & Value
- 「なぜこの機能が必要なのですか?」
- 「この機能がないと、どんな問題が起きますか?」
- 「この機能で、ビジネスにどんな影響がありますか?」
- 「成功をどのように測定しますか?」
- 「この機能の優先度が高い理由は?」

### User & Context
- 「この機能を使うのは誰ですか?」
- 「どんな状況で使いますか?」
- 「どのくらいの頻度で使われますか?」
- 「ユーザーはどんな目的で使いますか?」
- 「現在、ユーザーはこの問題をどう解決していますか?」

### Problem Understanding
- 「現在の最大の課題は何ですか?」
- 「この問題が起きる原因は何ですか?」
- 「どんなデータがこの問題を示していますか?」
- 「この問題の影響を受けるのは誰ですか?」
- 「この問題はいつから発生していますか?」

## Clarification Questions (明確化の質問)

### Concrete Examples
- 「具体的な例を教えていただけますか?」
- 「理想的なケースでは、どう動作しますか?」
- 「最悪のケースでは、どうなりますか?」
- 「典型的な使用例を教えてください」
- 「例外的なケースはありますか?」

### Definition & Scope
- 「『〜〜』とは、具体的にどういう意味ですか?」
- 「この機能に含めるもの、含めないものは?」
- 「どこまでが対象範囲ですか?」
- 「境界条件は何ですか?」
- 「この機能と他の機能の関係は?」

### Assumptions & Constraints
- 「どんな前提条件がありますか?」
- 「技術的な制約はありますか?」
- 「予算や時間の制約は?」
- 「セキュリティやプライバシーの要件は?」
- 「パフォーマンス要件は?」

## Validation Questions (検証の質問)

### Understanding Check
- 「私の理解は正しいですか?」
- 「つまり、〜〜という理解で合っていますか?」
- 「言い換えると、〜〜ということですか?」
- 「この仮説で抜けている点はありますか?」
- 「他に重要な観点はありますか?」

### Priority & Trade-offs
- 「複数の要求がある場合、優先順位は?」
- 「時間が限られている場合、何を諦めますか?」
- 「AとB、どちらがより重要ですか?」
- 「なぜAよりBを優先するのですか?」
- 「リリース後、何を追加する予定ですか?」

### Success Criteria
- 「何をもって成功と言えますか?」
- 「どんな指標で効果を測りますか?」
- 「どのくらいの改善を期待していますか?」
- 「リリース後、何をモニタリングしますか?」
- 「ユーザーからどんなフィードバックを期待しますか?」

## Exploration Questions (探索の質問)

### Alternative Solutions
- 「他の解決方法は考えましたか?」
- 「この機能以外に、問題を解決する方法は?」
- 「既存の機能で対応できませんか?」
- 「シンプルな代替案はありますか?」
- 「段階的にリリースする方法は?」

### Future Vision
- 「将来的に、この機能はどう進化しますか?」
- 「次のフェーズで追加したい機能は?」
- 「長期的なロードマップは?」
- 「この機能が他の計画に与える影響は?」
- 「拡張性をどう考えていますか?」

### Stakeholder Perspectives
- 「他のステークホルダーの意見は?」
- 「ユーザー以外に影響を受ける人は?」
- 「経営層の期待は?」
- 「開発チームの懸念は?」
- 「運用チームの準備は?」

## 5W1H Template

### What
- 「具体的に何を実現したいですか?」
- 「何が問題ですか?」
- 「何が変わりますか?」

### Who
- 「誰が使いますか?」
- 「誰が影響を受けますか?」
- 「誰が決定権を持っていますか?」

### When
- 「いつ使いますか?」
- 「いつまでに必要ですか?」
- 「いつリリースしますか?」

### Where
- 「どこで使いますか?」
- 「どのプラットフォームですか?」
- 「どの画面で使いますか?」

### Why
- 「なぜ必要ですか?」
- 「なぜ今なのですか?」
- 「なぜ他の方法ではダメなのですか?」

### How
- 「どのように動作すべきですか?」
- 「どのように測定しますか?」
- 「どのように実現しますか?」

## Anti-Patterns to Avoid

### ❌ Leading Questions (誘導質問)
```
Bad: 「Elasticsearchを使えば解決しますよね?」
Good: 「どんな検索機能が必要ですか?」
```

### ❌ Multiple Questions at Once
```
Bad: 「誰が使って、いつ使って、どう使いますか?」
Good: 「誰が使いますか?」→ 回答後 → 「いつ使いますか?」
```

### ❌ Closed Questions Only
```
Bad: 「検索機能は必要ですか?」(Yes/No)
Good: 「ユーザーはどのように商品を見つけますか?」
```

### ❌ Technical Jargon
```
Bad: 「RESTful APIで良いですか?」
Good: 「システム間でどんなデータをやり取りしますか?」
```

### ❌ Assumption-Based Questions
```
Bad: 「当然モバイル対応ですよね?」
Good: 「どのデバイスで使われますか?」
```

## Conversation Flow Example

```
[Phase 1: Open Discovery]
Q: 「何を実現したいですか?」
A: 「検索機能が欲しい」

[Phase 2: Why Deep Dive]
Q: 「なぜ検索機能が必要なのですか?」
A: 「商品が見つからない」

Q: 「なぜ商品が見つからないのですか?」
A: 「商品が多すぎる」

Q: 「具体的には何商品くらいですか?」
A: 「500商品以上」

[Phase 3: Concrete Example]
Q: 「具体的に、ユーザーはどんな商品を探していますか?」
A: 「例えば、前に買ったノートパソコンの周辺機器」

[Phase 4: Validation]
Q: 「つまり、過去の購入履歴から関連商品を見つけたい、
    という理解で合っていますか?」
A: 「そうです」

[Phase 5: Hypothesis]
Q: 「私の理解では、検索機能 + 購入履歴連携が必要ですね?」
A: 「はい、その通りです」
```
